## 🔹 What is Machine Learning in NLP?

Machine Learning allows computers to **learn patterns from data** and make decisions.

👉 In NLP:

* Input → Text (converted to numbers)
* Output → Prediction (label, sentiment, category, etc.)

---

## 🔹 A Simple Example

### Problem:

Classify emails as:

* Spam
* Not Spam

### Process:

1. Convert text → numbers (TF-IDF)
2. Train model → learn patterns
3. Predict → classify new emails

👉 Like a student learning from examples and then solving new questions.

---

## 🔹 Types of Machine Learning in NLP

---

### 🔸 1. Supervised Learning

👉 The model learns from **labeled data**

### Example:

| Text          | Label    |
| ------------- | -------- |
| “I love this” | Positive |
| “This is bad” | Negative |

👉 The model learns:

* “love” → positive
* “bad” → negative

---

### 🔸 2. Unsupervised Learning

👉 No labels are given

👉 The model finds patterns on its own

### Example:

* Group similar documents together
* Topic modeling

👉 Like grouping similar books without knowing their category

---

## 🔹 Common ML Algorithms for NLP

---

## 🔸 1. Naive Bayes

👉 Based on probability

👉 Very popular for text classification

---

### Example:

* “Win money now!!!” → Spam
* “Meeting at 5pm” → Not Spam

👉 It calculates probability of words appearing in each class

---

### Intuition

Think like this:

* If a message contains “win”, “money”
  👉 High chance of spam

---

### Advantages

* Fast
* Works well for text

---

## 🔸 2. Logistic Regression

👉 Used for classification (yes/no type problems)

---

### Example:

* Sentiment:

  * Positive
  * Negative

👉 It learns a boundary between classes

---

### Intuition

Imagine a line separating:

* Positive reviews
* Negative reviews

---

## 🔸 3. Support Vector Machine (SVM)

👉 Finds the **best boundary** between classes

---

### Example:

* Classifying news:

  * Sports
  * Politics

👉 SVM finds the clearest separation

---

### Intuition

Think of drawing the **widest gap** between two groups

---

## 🔸 4. Decision Trees

👉 Uses **if-else rules**

---

### Example:

* If message contains “offer” → Spam
* Else → Not Spam

---

### Intuition

Like a flowchart decision system

---

## 🔹 Model Evaluation

How do we know if the model is good?

---

### 🔸 Accuracy

👉 Percentage of correct predictions

Example:

* 90 correct out of 100 → 90% accuracy

---

### 🔸 Precision & Recall (Basic Idea)

* Precision → How correct the positive predictions are
* Recall → How many actual positives we found

👉 Important in problems like spam detection

---

## 🔹 Simple Python Example

```python id="ml-nlp-example"
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.naive_bayes import MultinomialNB

# Sample data
texts = ["I love this movie", "This is bad", "Amazing film", "Terrible movie"]
labels = ["Positive", "Negative", "Positive", "Negative"]

# Convert text to numbers
vectorizer = CountVectorizer()
X = vectorizer.fit_transform(texts)

# Train model
model = MultinomialNB()
model.fit(X, labels)

# Test
test = vectorizer.transform(["I love this"])
print(model.predict(test))
```

👉 Output:

```
['Positive']
```

## 🔹 Limitations of Basic ML in NLP

* Cannot understand deep meaning
* Struggles with context
* Depends heavily on feature engineering

👉 Example:

* “not bad” → might be misclassified as negative

* Common algorithms:

* Workflow:

  * Text → preprocess → features → model → prediction
