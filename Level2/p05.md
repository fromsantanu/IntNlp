# 📘 Chapter 2.5 — Evaluation Metrics in NLP

---

After building an NLP model, an important question arises:

👉 **How do we know if the model is good or not?**

This is where **evaluation metrics** come in.

👉 They help us measure **how well our model is performing**

---

## 🔹 Why Evaluation Metrics are Important

Imagine a student giving an exam:

* Without marks → no idea how well they performed
* With marks → clear understanding

👉 Similarly:

* Model predictions → answers
* Evaluation metrics → marks

---

## 🔹 Types of NLP Evaluation

Depending on the task, evaluation changes:

| Task            | Metric                                |
| --------------- | ------------------------------------- |
| Classification  | Accuracy, Precision, Recall, F1-score |
| Language Models | Perplexity                            |
| Translation     | BLEU Score                            |
| Summarization   | ROUGE Score                           |

---

# 🔸 1. Accuracy

## 🔹 Definition

👉 Accuracy = How many predictions are correct

$Accuracy = \frac{\text{Correct Predictions}}{\text{Total Predictions}}$

---

## 🔹 Example

* Total emails = 100
* Correct predictions = 90

👉 Accuracy = 90%

---

## 🔹 Limitation

Accuracy can be misleading when data is imbalanced.

### Example:

* 95 spam, 5 non-spam
* Model predicts everything as spam

👉 Accuracy = 95% ❌ (but useless)

---

# 🔸 2. Precision

## 🔹 Definition

👉 Out of predicted positives, how many are correct?

$Precision = \frac{TP}{TP + FP}$

---

## 🔹 Example

Spam detection:

* Predicted spam = 50
* Actually spam = 40

👉 Precision = 40 / 50 = 80%

---

## 🔹 When Important?

👉 When false positives are costly
Example:

* Marking important email as spam

---

# 🔸 3. Recall

## 🔹 Definition

👉 Out of actual positives, how many did we detect?

$Recall = \frac{TP}{TP + FN}$

---

## 🔹 Example

* Total spam = 100
* Detected spam = 80

👉 Recall = 80%

---

## 🔹 When Important?

👉 When missing positives is costly
Example:

* Detecting diseases

---

# 🔸 4. F1 Score

## 🔹 Definition

👉 Balance between Precision and Recall

$F1 = 2 \cdot \frac{Precision \cdot Recall}{Precision + Recall}$

---

## 🔹 Why Needed?

* Precision alone is not enough
* Recall alone is not enough

👉 F1 gives a balanced measure

---

# 🔸 5. Confusion Matrix

A table showing prediction results.

---

## 🔹 Structure:

|                 | Predicted Positive | Predicted Negative |
| --------------- | ------------------ | ------------------ |
| Actual Positive | TP                 | FN                 |
| Actual Negative | FP                 | TN                 |

---

## 🔹 Meaning:

* TP → Correct positive
* TN → Correct negative
* FP → Wrong positive
* FN → Missed positive

# 🔸 6. Perplexity (Language Models)

👉 Used for evaluating **language models**

---

## 🔹 Idea

👉 Measures how well a model predicts a sentence

* Low perplexity → good
* High perplexity → poor

---

## 🔹 Intuition

Think of it like:

* If model is “confused” → high perplexity
* If confident → low perplexity

---

# 🔸 7. BLEU Score (Machine Translation)

👉 Measures quality of translated text

---

## 🔹 Idea

Compare:

* Machine translation
* Human reference translation

👉 Higher BLEU → better translation

---

## 🔹 Example

* Reference: “The cat is on the mat”
* Model: “The cat is on mat”

👉 Similar → high BLEU score

---

# 🔸 8. ROUGE Score (Summarization)

👉 Measures quality of summaries

---

## 🔹 Idea

Compare overlap between:

* Generated summary
* Reference summary

👉 Higher overlap → better summary

---

## 🔹 Real-Life Analogy

Think of evaluating answers in an exam:

* Accuracy → total marks
* Precision → correctness of selected answers
* Recall → how many correct answers attempted
* F1 → overall balance

---

## 🔹 When to Use Which Metric?

| Situation           | Best Metric |
| ------------------- | ----------- |
| Balanced dataset    | Accuracy    |
| Avoid false alarms  | Precision   |
| Avoid missing cases | Recall      |
| Need balance        | F1 Score    |
| Language prediction | Perplexity  |
| Translation         | BLEU        |
| Summarization       | ROUGE       |

---

## 🔹 Summary

* Evaluation metrics measure **model performance**
* Key metrics:

  * Accuracy
  * Precision
  * Recall
  * F1 Score
* Special metrics:

  * Perplexity (language models)
  * BLEU (translation)
  * ROUGE (summarization)
* Choosing the right metric is very important
